# 未実装機能一覧と実装方針

本ドキュメントは、仕様書に記載されているが現時点で未実装の機能について、実装の方針・手順・技術的な考慮事項をまとめたものである。

---

## 1. オーディオ（BGM / 効果音）

### 概要
基本設計・詳細設計の両方で定義されているが、完全に未実装。設定UIにも「未実装」の注記がある状態。

### 実装方針

#### 1.1 サウンドエンジン
- Phaser の **Sound Manager**（Web Audio API バックエンド）を利用する
- RunScene の `preload()` でオーディオアセットを読み込む

#### 1.2 必要なサウンドアセット

| 種別 | ファイル名例 | 再生タイミング |
|------|------------|--------------|
| BGM | `bgm_title.ogg` | タイトル画面表示中 |
| BGM | `bgm_game.ogg` | ゲームプレイ中（ウェーブ進行で変化してもよい） |
| SE | `se_hit.ogg` | ボールがブロックに衝突 |
| SE | `se_paddle.ogg` | ボールがパドルに衝突 |
| SE | `se_destroy.ogg` | ブロック破壊 |
| SE | `se_item_get.ogg` | ドロップアイテム取得 |
| SE | `se_miss.ogg` | ボール落下（ミス） |
| SE | `se_skill.ogg` | スキル発動 |
| SE | `se_wave_clear.ogg` | ウェーブクリア |
| SE | `se_gameover.ogg` | ゲームオーバー |
| SE | `se_combo.ogg` | コンボ継続（ピッチ変化で連鎖感を演出） |
| SE | `se_hurry.ogg` | パニックモード突入 |

#### 1.3 アセット配置
```
public/assets/sounds/
  bgm/
    bgm_title.ogg
    bgm_game.ogg
  se/
    se_hit.ogg
    se_paddle.ogg
    ...
```

#### 1.4 設定項目の追加

`settingsStore.ts` の `GameSettings` に以下を追加：

```typescript
interface GameSettings {
  // ...既存フィールド
  volumeBGM: number;   // 0〜100（既定: 70）
  volumeSE: number;    // 0〜100（既定: 80）
}
```

#### 1.5 設定UI
- `index.html` の設定オーバーレイにスライダー2本（BGM / SE）を追加
- `main.ts` で値の変更を `GameSettings` に反映し、`localStorage` に保存
- 変更は即時プレビュー（スライダー操作中にテスト音を鳴らす）

#### 1.6 RunScene への組み込み
- `create()` で BGM を開始。一時停止時は `scene.sound.pauseAll()`、復帰時は `scene.sound.resumeAll()`
- 衝突コールバック内で SE を `play()` する
- コンボ SE はコンボ数に応じてピッチを上げる（`detune` パラメータ）
- Reduced Motion 有効時は BGM のみに絞り、SE の演出を控えめにする選択肢もあり

#### 1.7 実装順序
1. アセット調達・配置（フリー素材 or 自作）
2. `settingsStore.ts` にフィールド追加 + デフォルト値設定
3. RunScene の `preload()` にオーディオロード追加
4. RunScene の各イベントに SE 再生を組み込み
5. 設定UIにスライダー追加
6. 一時停止・ゲームオーバー時のサウンド制御

## 2. テーマ切替（ダーク / ライト）

### 概要
詳細設計で「画面明度/テーマ切替（ダーク/ライト選択）」として定義。現在は `root.dataset.theme = 'light'` が固定されており、実質的にネオン系テーマのみ。

### 実装方針

#### 2.1 CSS 変数化
- `ui.css` の色指定をすべて CSS カスタムプロパティ（デザイントークン）に置き換える
- `[data-theme="light"]` と `[data-theme="dark"]` で値を切り替える

```css
:root, [data-theme="light"] {
  --color-bg: #F8FAFC;
  --color-surface: #FFFFFF;
  --color-text: #0F172A;
  --color-text-sub: #475569;
  --color-primary: #2563EB;
  --color-accent: #0F766E;
  --color-border: #CBD5E1;
  /* ... */
}
[data-theme="dark"] {
  --color-bg: #0F172A;
  --color-surface: #1E293B;
  --color-text: #F1F5F9;
  --color-text-sub: #94A3B8;
  --color-primary: #60A5FA;
  --color-accent: #2DD4BF;
  --color-border: #334155;
  /* ... */
}
```

#### 2.2 設定項目の追加

`settingsStore.ts` の `GameSettings` に追加：

```typescript
interface GameSettings {
  // ...既存フィールド
  theme: 'light' | 'dark';  // 既定: 'light'
}
```

#### 2.3 設定UI
- 設定オーバーレイにトグルスイッチまたはセレクトを追加
- 変更時に `document.documentElement.dataset.theme` を即時切替
- `applySettingsToDocument()` にテーマ反映ロジックを追加

#### 2.4 Phaser Canvas 側
- RunScene 内の描画色（HUD、背景、テキスト等）もテーマに応じた色マップを参照する
- `create()` 時に設定からテーマカラーを読み取り、各描画オブジェクトに適用

#### 2.5 実装順序
1. `ui.css` の全色指定を CSS 変数に移行
2. light / dark の変数セットを定義
3. `settingsStore.ts` にフィールド追加
4. `applySettingsToDocument()` を拡張
5. 設定UIにトグル追加
6. RunScene のHUD/背景色を設定連動に変更

## 3. テキスト倍率設定

### 概要
詳細設計で「テキスト倍率（100%/125%/150%など）」として定義。現在は `--ui-scale: 1` 固定。

### 実装方針

#### 3.1 CSS 変数による倍率適用
- 既に `--ui-scale` カスタムプロパティが設定されているので、これを活用する
- HTMLシェル側のフォントサイズ・余白に `calc()` で `--ui-scale` を乗算する

```css
.ui-text { font-size: calc(16px * var(--ui-scale)); }
.ui-heading { font-size: calc(24px * var(--ui-scale)); }
```

#### 3.2 設定項目の追加

```typescript
interface GameSettings {
  // ...既存フィールド
  textScale: number;  // 1.0 / 1.25 / 1.5（既定: 1.0）
}
```

#### 3.3 設定UI
- 設定オーバーレイに3段階のセレクトまたはラジオボタンを追加（100% / 125% / 150%）
- 変更は即時プレビュー

#### 3.4 適用
- `applySettingsToDocument()` で `root.style.setProperty('--ui-scale', String(s.textScale))` を設定
- Phaser Canvas 内のHUDテキストサイズには直接影響させない（Canvas内は固定サイズが望ましい）

#### 3.5 実装順序
1. `ui.css` のフォントサイズを `calc()` + `--ui-scale` で書き直す
2. `settingsStore.ts` にフィールド追加
3. `applySettingsToDocument()` を拡張
4. 設定UIにセレクト追加
5. レイアウト崩れが起きないかを各倍率でテスト

## 4. ビルド/強化選択（ウェーブ間の選択UI）

### 概要
要件定義 §16 の拡張予定として「ビルド（強化選択）拡張」が記載。基本設計の P0 未解決事項で「初期版は未実装」と決定済み。ローグライク要素の戦略性を高める中核機能。

### 実装方針

#### 4.1 ゲームデザイン

ウェーブクリア時（全ブロック破壊 or タイマー終了後）に、次ウェーブ開始前に **3択のビルドカード** を提示し、プレイヤーが1つを選択する。

| カテゴリ | 例 |
|----------|---|
| パドル強化 | パドル基本幅 +10%（永続） |
| ボール強化 | ボール基本速度 -5%（永続） |
| スキル強化 | スキルゲージ蓄積速度 +20% |
| アイテム強化 | バフアイテムの効果時間 +30% |
| 防御強化 | 残機 +1（上限5） |
| リスク/リターン | ボール速度 +15% だがスコア倍率 ×1.5 |

#### 4.2 UI設計
- ウェーブクリア直後に DOM オーバーレイ（`#overlay-build-select`）を表示
- 3枚のカード（アイコン・タイトル・説明文・バフ/デバフタグ）を中央に配置
- キーボード（←→ + Enter）/ クリックで選択
- 選択後に次ウェーブのカウントダウンを開始

#### 4.3 データ構造

```typescript
interface BuildCard {
  id: string;
  label: string;
  description: string;
  icon: string;
  rarity: 'common' | 'rare' | 'epic';
  apply: (state: RunSceneState) => void;
}
```

- ビルドカードプールを定義し、ウェーブ数・既選択カードに応じて抽選ロジックを組む
- 選択済みビルドは RunScene のステートに累積し、HUD に一覧表示

#### 4.4 RunScene への組み込み
- `advanceWave()` 内で、次ウェーブ生成前にビルド選択イベントを発火
- DOM 側で選択を受け取ったら、効果を適用して `resumeFromBuildSelect()` を呼ぶ
- 一時停止オーバーレイと同様の仕組み（カスタムイベント連携）で実装

#### 4.5 実装順序
1. ビルドカードのデータ定義（カード一覧、効果関数）
2. 抽選ロジック実装
3. DOM オーバーレイ UI 追加（index.html + main.ts）
4. RunScene のウェーブ遷移にフック追加
5. 選択済みビルドのHUD表示
6. バランス調整（カード効果値のチューニング）

## 優先度と依存関係

| 機能 | 優先度 | 工数目安 | 依存 |
|------|--------|---------|------|
| オーディオ（BGM/SE） | 中 | 中（アセット調達含む） | アセット準備が先行条件 |
| テーマ切替 | 低 | 小〜中 | CSS変数化が前提 |
| テキスト倍率 | 低 | 小 | CSS変数化が前提（テーマ切替と共通作業） |
| ビルド/強化選択 | 低（将来拡張） | 大 | ゲームバランスの設計が先行条件 |

### 推奨実装順序

1. **テーマ切替 + テキスト倍率**（CSS変数化を共通作業としてまとめて実施）
2. **オーディオ**（アセットが揃い次第）
3. **ビルド/強化選択**（ゲームデザインの詳細を詰めた後）
