# ブロック崩し（ローグライク型）技術スタック定義

## 概要
本プロジェクトは、透過画像をベースにしたステージ生成と、ローグライク要素（ランダム変形・難易度上昇）を組み合わせたブロック崩しゲームを開発する。

開発は Cursor を前提とし、コードベースでの分割実装を行う。
また、最終的には Windows 向けにインストーラー形式で配布する。

## プロジェクト構成

```
breaking_blocks/
  src/
    main.ts                         # アプリ起動、画面遷移、DOMオーバーレイ管理
    game/
      scenes/
        RunScene.ts                 # ゲームプレイの全ロジック（Phaser唯一のScene）
      ImageStageGenerator.ts        # 透過PNG → グリッドブロック生成
      DifficultyProfile.ts          # 5段階難易度の定義
      settingsStore.ts              # 設定の永続化（localStorage）
      highscores.ts                 # ハイスコア保存・読込（localStorage、Top 20）
      keyBindings.ts                # キーバインド定義・UIラベル変換
      uiI18n.ts                     # 日本語/英語の多言語文字列
    types/
      gameModel.ts                  # 型定義の再エクスポート
    app/
      logger.ts                     # インメモリイベントログ、テキストエクスポート
    styles/
      ui.css                        # HTML Shell のスタイル
    vite-env.d.ts
  public/
    assets/
      blocks/                       # シルエット元画像（block1〜3.png）
  src-tauri/                        # Tauri 2 デスクトップ設定
  仕様書/                            # 設計ドキュメント
  index.html                        # UIシェル（全画面・モーダルのDOM定義）
  package.json
  pnpm-lock.yaml
  tsconfig.json
  vite.config.ts
  Makefile
```

### アーキテクチャ方針

Phaserシーンは **RunScene のみ** とし、タイトル・難易度選択・設定・一時停止・リザルト・ハイスコア等のUI画面は **DOMオーバーレイ（index.html + main.ts）** で実装する。この構成により、メニュー系UIのアクセシビリティ（キーボード操作、スクリーンリーダー対応）をcanvasに依存せず確保できる。

## 設計方針（重要）

### 1. 画像の扱い
- 1080×1080画像を元データとする
- 内部で縮小して使用（例：128px）
- グリッド単位でブロック生成

---

### 2. ブロック生成
- ピクセル単位では扱わない
- セル単位で配置
- 密度に応じて生成

---

### 3. ローグライク要素
- ステージはランダム生成
- セルの削除・強化・変形
- ウェーブごとに難易度上昇

---

### 4. 難易度設計
- 細かさで難易度を上げない
- 以下で調整する
  - 密度
  - 耐久
  - ギミック
  - ボール速度

---

### 5. 拡張性
- システムごとに責務分割
- Cursorで単位実装可能な構造

---
